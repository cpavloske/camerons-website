Building an Animated Star Field with Canvas

February 20, 2026

One of my favorite background effects for dark-themed websites is a subtle animated star field. It adds depth and atmosphere without being distracting. In this post, I'll walk through how I built one using the HTML Canvas API and vanilla JavaScript.

Setting Up the Canvas

The first step is creating a full-screen canvas element that sits behind all of our content. We use position: fixed and pointer-events: none so it stays in place and doesn't interfere with clicks.

<canvas id="star-canvas"></canvas>

<style>
  #star-canvas {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    height: 100%;
    z-index: 0;
    pointer-events: none;
  }
</style>

Generating the Stars

Each star is an object with random position, size, opacity, and drift speed. We store them in an array and regenerate whenever the window resizes.

const STAR_COUNT = 180;
let stars = [];

function createStars() {
  stars = [];
  for (let i = 0; i < STAR_COUNT; i++) {
    stars.push({
      x: Math.random() * canvas.width,
      y: Math.random() * canvas.height,
      radius: Math.random() * 1.4 + 0.3,
      alpha: Math.random() * 0.6 + 0.2,
      drift: Math.random() * 0.15 + 0.02,
    });
  }
}

The Animation Loop

We use requestAnimationFrame for a smooth 60fps loop. Each frame, we clear the canvas, draw every star with a gentle twinkling effect using a sine wave, and move them slowly upward to simulate drifting through space.

function draw(time) {
  ctx.clearRect(0, 0, canvas.width, canvas.height);

  for (const star of stars) {
    const twinkle = Math.sin(time * 0.005) * 0.3 + 0.7;
    ctx.beginPath();
    ctx.arc(star.x, star.y, star.radius, 0, Math.PI * 2);
    ctx.fillStyle = `rgba(200, 220, 255, ${star.alpha * twinkle})`;
    ctx.fill();

    star.y -= star.drift;
    if (star.y < -5) {
      star.y = canvas.height + 5;
      star.x = Math.random() * canvas.width;
    }
  }

  requestAnimationFrame(draw);
}

Performance Considerations

Canvas rendering is very efficient for this kind of particle effect. A few hundred circles drawn per frame is trivial for modern GPUs. The key is to avoid creating garbage by reusing star objects instead of allocating new ones each frame.

Also make sure you're listening for the resize event to update the canvas dimensions — otherwise your stars will render at the wrong resolution on window resize.

Wrapping Up

That's all there is to it. A few dozen lines of JavaScript gives you a beautiful, performant animated background that works in every modern browser. No libraries, no build tools — just the platform.
